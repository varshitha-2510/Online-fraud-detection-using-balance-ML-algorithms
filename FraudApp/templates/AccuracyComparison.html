Accuracy Improvement Across Fraud Detection Stages

📌 System Accuracy Stages
1️⃣ Before SMOTE – Traditional ML
The dataset is highly imbalanced. The model mainly learns non-fraud patterns, resulting in lower fraud detection accuracy.
2️⃣ After SMOTE – Balanced ML Model
SMOTE generates synthetic fraud samples and balances the dataset. This improves learning of minority fraud class.
3️⃣ Hybrid Rule + ML – Real-Time Detection
Combines domain rules (amount drain, receiver risk, velocity rules) with ML probability output, giving maximum accuracy.
📈 Accuracy Formula
Accuracy = (TP + TN) / (TP + TN + FP + FN) × 100

Where:
TP – Fraud correctly detected
TN – Normal transaction correctly classified
FP – Normal transaction wrongly flagged as fraud
FN – Fraud transaction missed by system
🏁 Conclusion
The graph clearly proves that applying SMOTE and then integrating a Hybrid Rule + Machine Learning real-time engine significantly improves fraud detection accuracy.
